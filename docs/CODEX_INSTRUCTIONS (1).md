### アイコン画像の使用方針

`アイコン.png` は8個のアイコンが2行4列で並んでいる。
個別切り出しが難しい場合は、各フィーチャーカードのアイコン箇所に `<img src="../img/アイコン.png" ...>` を使わず、代わりにUnicode絵文字（🗓 📍 👥 ✉）か インラインSVGで代替し、`アイコン.png` はfooterの将棋盤アイコンなど単一用途に限定して使用してよい。

### デザイントークン

```css
:root {
  --color-primary: #3d6b45;      /* 深緑 */
  --color-primary-dark: #2d5235; /* 濃緑（hover） */
  --color-accent: #8b6914;       /* 金茶（将棋駒色） */
  --color-bg: #faf7f2;           /* 温かいオフホワイト */
  --color-bg-card: #ffffff;
  --color-text: #2c2c2c;
  --color-text-muted: #666666;
  --color-border: #e8e0d4;
  --font-main: 'Noto Serif JP', 'Yu Mincho', 'YuMincho', serif;
  --font-ui: 'Noto Sans JP', 'Hiragino Sans', sans-serif;
  --shadow-card: 0 2px 12px rgba(0,0,0,0.08);
  --radius: 8px;
}
```

### 必須スタイル要件

- **設計方針**: **モバイルファースト**（スマホ9割想定）。PCは後からメディアクエリで拡張する方向で書くこと
- **ベースフォントサイズ**: `font-size: 16px` を最小とし、本文は`1rem`、見出しは`1.4rem`以上。シニア層が読みやすいよう小さくしすぎない
- **タップターゲット**: ボタン・リンク・ナビ項目は全て `min-height: 48px` 以上確保（指で押しやすいサイズ）
- **ヘッダー**: スマホでは最初からハンバーガーメニュー表示。PCでは横並びナビ
- **ヒーローセクション**: スマホでは情報カード（活動日・会場・参加費）をテキスト下に縦積み表示。PCでは左テキスト＋右カードの横並び
- **フィーチャー3列**: スマホでは1列縦積み。PCでは3列グリッド
- **観光スポットカード**: スマホでは2列（`minmax(150px, 1fr)`）。PCでは4列
- **ブレークポイント**: `768px`のみ（シンプルに1点で切り替え）
- **電話番号リンク**: `<a href="tel:...">` を大きめボタン風にスタイリング（スマホからワンタップ発信）
- **Google Maps embed**: スマホで `height: 300px`、PCで `height: 450px`
- **フォント**: Google Fonts `Noto Serif JP` + `Noto Sans JP` をheadでロード（`@import`は使わず`<link>`タグで）
- **行間**: `line-height: 1.8`（日本語・シニア向けに広めに）

## 実装上の注意事項

1. **img/ フォルダのパス**: index.htmlからは `img/ファイル名`、pages/ 配下のHTMLからは `../img/ファイル名` で参照すること
2. **ChatGPT生成画像のファイル名**: `ChatGPT_Image_2026年6月26日_16_29_19.png` のように日本語・スペースを含む。HTMLの`src`属性ではそのまま記述してよいが、必要であれば英語ファイル名にリネームしてもよい（`gallery-tournament.png` など）
3. **予定表例.png の扱い**: 画像をそのままサイトに貼るのではなく、記載内容（日付・文言）をテキストとして転記し、掲示板風カードUIとして activity.html に実装すること
4. **画像の`alt`属性**: 必ず日本語で記述すること（例: `alt="大子連合将棋大会の様子"`）
5. **画像が未配置の場合**: `<img>`タグのsrcを `images/placeholder.jpg` とし、背景色 `#e8e0d4` のdivでプレースホルダー表示
6. **フォント**: CDNの読み込みに失敗した場合のフォールバックをCSSに必ず記述
7. **ナビゲーション**: 各ページから `index.html` への相対パスで `../index.html`、同階層は `./activity.html`
